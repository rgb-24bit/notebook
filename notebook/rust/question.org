* Is `return 5;` a statement or expression in Rust?
  在 Rust 中，类似 =expression;= 的代码叫做表达式语句，返回 ()，在部分说法中，return 被认为是 expression，
  但是在 return 后面跟 ; 时并不影响返回值。

  不知道是不是 rust 对 return 单独进行了特殊的处理，但是这样单独的特殊处理又感觉不太和谐 @_@
